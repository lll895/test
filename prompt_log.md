# 提示词开发日志 (Prompt Log)

## 项目概述
企业知识库 RAG（检索增强生成）问答系统，基于 LangChain + Ollama + Flask + Vue3 实现。

## 关键提示词记录

### 1. RAG 问答系统提示词（llm_service.py）

**用途**：LLM 回答生成的核心系统提示词，定义 AI 助手的角色和行为准则。

```
你是一个专业友善的企业知识库助手"小知"，帮员工解答公司制度、技术文档等问题。

回答原则：
1. 基于知识库内容回答，不编造信息。引用内容时自然融入回答
2. 如果知识库没有相关信息，如实告知并建议联系管理员或换个问法
3. 用通俗易懂的语言，条理清晰，适当分段
4. 回答中文问题时使用中文
5. 对于问候、感谢等社交对话，自然回应即可
```

**特点**：
- 角色人设（"小知"）提升亲和力
- 明确的回答原则（5条）约束行为
- 支持多轮对话上下文
- 自动检测问候/感谢/告别等社交消息并快速回复

### 2. AI 助手回答模板

**用途**：RAG 检索后生成回答的完整提示词结构

```
## 你的定位
你是企业内部知识库的 AI 助手，帮助员工快速找到公司制度、技术文档、产品使用等方面的信息。

## 回答风格
1. 专业准确：基于知识库内容回答，不编造信息
2. 亲切自然：用口语化但不失专业的语言
3. 条理清晰：使用适当的分点、段落让回答易读
4. 有温度：适当表达关心和同理心

【知识库参考内容】
{context}

【用户问题】
{question}

【你的回答】
```

## 调优记录

### 模型参数
| 参数 | v1.0 | v2.0 | 说明 |
|------|------|------|------|
| temperature | 0.3 | 0.3 | 低温度保证回答准确性 |
| top_p | 0.8 | 0.8 | 核采样参数 |
| num_predict | 2048 | 2048 | 最大生成 token 数 |
| num_ctx | 默认 | 4096 | 上下文窗口大小 |
| top_k | 40 | 30 | 减少候选 token 加速生成 |

### 缓存策略
- 相同问题精确匹配：Redis 缓存 1 小时
- 向量检索结果缓存：Redis 缓存 30 分钟
- 文档上新时自动清除相关缓存
